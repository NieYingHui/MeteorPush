# 快速上手指南

> 目标：30 分钟内编译、启动全部服务、发一条消息看到效果。

## 环境要求

- **操作系统**: Ubuntu 22.04 
- **编译器**: g++ 11+ 或 clang++ 14+（支持 C++17）
- **CMake**: 3.14+

---
 

## 第 1 步：启动中间件（10 分钟）

### Redis

```bash
sudo apt-get install -y redis-server
sudo systemctl start redis-server
# 验证
redis-cli ping    # 应返回 PONG
```

### MySQL

```bash
sudo apt-get install -y mysql-server
sudo systemctl start mysql

# 初始化数据库
mysql -u root -p < sql/schema.sql
```

如果 MySQL 需要密码，修改 `conf/logic.conf` 和 `conf/job.conf` 中的 `mysql_password`。

### Kafka

推荐 KRaft 模式（无需 ZooKeeper），详见 `kafka安装.md`。简要步骤：

```bash
# 启动
bin/kafka-server-start.sh -daemon config/kraft/server.properties

# 创建 topic
bin/kafka-topics.sh --create --topic push_to_comet --partitions 4 --replication-factor 1 \
    --bootstrap-server localhost:9092
bin/kafka-topics.sh --create --topic broadcast_task --partitions 1 --replication-factor 1 \
    --bootstrap-server localhost:9092
```

## 第 2 步：编译（3 分钟）

```bash
cd  meteor_push/06

mkdir -p build && cd build
cmake .. -DCMAKE_BUILD_TYPE=Release
make -j$(nproc)
```

编译成功后会生成：
- `build/logic/logic_server`
- `build/comet/comet_server`
- `build/job/job_server`
- `build/web_demo/web_demo_server`（可选）

---

## 第 3 步：启动服务（2 分钟）

**启动顺序**: Logic → Comet → Job（Logic 先启动，因为 Comet 要连 Logic 的 gRPC）

```bash
# 终端 1: 启动 Logic
cd /path/to/meteor-push
./build/logic/logic_server

# 终端 2: 启动 Comet
./build/comet/comet_server

# 终端 3: 启动 Job
./build/job/job_server

# 终端 4: 启动 Web Demo（可选）
./build/web_demo/web_demo_server
```

或者用脚本一键启动：

```bash
bash scripts/start_all.sh
```

---

## 第 4 步：发一条消息（2 分钟）

### 方法 A：使用 Web Demo

1. 打开浏览器，访问 `http://localhost:9080`，注意： （单独启动web页面服务默认是9001， 一键启动时设置的9080）
2. 左侧面板（User A）：点击"快速登录"
3. 右侧面板（User B）：点击"快速登录"
4. 选择"单聊"模式
5. 在 User A 的输入框输入消息，点击发送
6. 观察 User B 是否收到消息

### 方法 B：使用 curl + wscat

```bash
# 注册两个用户
curl -X POST http://localhost:9101/api/register \
  -H 'Content-Type: application/json' \
  -d '{"account":"alice","password":"123456","name":"Alice"}'

curl -X POST http://localhost:9101/api/register \
  -H 'Content-Type: application/json' \
  -d '{"account":"bob","password":"123456","name":"Bob"}'

# 登录获取 token
curl -X POST http://localhost:9101/api/login \
  -H 'Content-Type: application/json' \
  -d '{"account":"alice","password":"123456"}'
# 记下返回的 token，如 "abc123"

curl -X POST http://localhost:9101/api/login \
  -H 'Content-Type: application/json' \
  -d '{"account":"bob","password":"123456"}'
# 记下 Bob 的 token，如 "def456"

# 用 wscat 建立 WebSocket 连接（需安装: npm install -g wscat）
# 终端 A（Alice）:
wscat -c "ws://localhost:9200/ws?token=abc123"

# 终端 B（Bob）:
wscat -c "ws://localhost:9200/ws?token=def456"

# 在 Alice 的终端发送消息（假设 Bob 的 user_id=2）:
> {"type":"single_chat","to_user_id":2,"content":"Hello Bob!","client_msg_id":"test1"}

# Bob 的终端应该收到消息
```

## 端口规划

| 服务 | 端口 | 用途 |
|------|------|------|
| Comet | 9200 | WebSocket 客户端连接 |
| Comet | 9205 | gRPC（接收 Job 推送） |
| Logic | 9100 | gRPC（接收 Comet 请求） |
| Logic | 9101 | HTTP REST API |
| Web Demo |  9080（单独启动默认是9001， 一键启动时设置的9080） | 静态文件服务 |

---

## 配置文件

| 文件 | 服务 | 关键配置 |
|------|------|---------|
| `conf/comet.conf` | Comet | listen_port, logic_grpc_target, use_grpc_stream |
| `conf/logic.conf` | Logic | kafka_brokers, redis_host, mysql_host, http_threads |
| `conf/job.conf` | Job | kafka_brokers, comet_targets, use_push_stream, mysql_host |

## 常见问题

### 编译报错 "grpc/grpc.h not found"

```bash
sudo apt-get install -y libgrpc++-dev protobuf-compiler-grpc
```

### Kafka 连接失败

确认 Kafka 是否启动：

```bash
# 检查 9092 端口
ss -tlnp | grep 9092
```

### MySQL 连接失败

检查 `conf/logic.conf` 和 `conf/job.conf` 中的 MySQL 配置（host/port/user/password）是否正确。

### Redis 连接失败

```bash
redis-cli ping    # 应返回 PONG
```

### WebSocket 连接失败

确认 Comet 和 Logic 都已启动。查看日志：

```bash
tail -f logs/comet.log
tail -f logs/logic.log
```
